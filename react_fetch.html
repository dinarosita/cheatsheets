response.json()

fetch(
    "...api url..."
).then(
    response => {return response.json)();}
).then(
    data => {...}
)
  • By default, fetch http request is executing GET method. So the second parameter, configuration object, is not needed. Simply first argument of API URL is enough.
  • Since fetch returns a promise, we can use then method to handle the respond.
  • The Response interface of the Fetch API represents the response to a request.response object is used as argument.
  • response.json() - Returns a promise that resolves with the result of parsing the response body text as JSON.
    • response.json() method to read the data. It gives us access to the data, automatically convert it from json to a plain js object.
    • response.json() will actually returns a promise as well, so we need to wait this promise to resolve. That's why it has to be coded as a return first, the data is processed further.

useState hook

  • We can't simply code return after the fetch codes though, because the return is not going to wait until the promises are fullfilled to execute.
  • Typically async and await can be used.However we can't use it in React, because it's turning the component to a promise (that's what async does), and the component is no longer valid React component. Because React component function must be synchronous and must not return a promise. They have to directly return a jsx.
    async function AllMeetupPage() {
    await fetch(...).then(...).then(...);
    return (...);
};
  • Instead we can return a temporary jsx code, for example a loading spinner, and once we have the response, we can update the returned jsx. To change what's visibe on the screen, use "state". We use state to choose what's visible on the screen when certain condition changed.
