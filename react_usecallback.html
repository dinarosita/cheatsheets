React useCallback

useCallback

This hook cache a function between rerendering. It is similar to useMemo, but in useMemo, it's the return value that gets cached. In the useCallback, it's the whole value.

In practise, it's a pretty rare occassion that caching the function is needed. Typically useMemo is what we need.

In the case the function will be called back using parameter given some other way, this can't be fulfilled by useMemo. Only useCallback can do this. However probably this is a very rare occurence. In most cases what we worry about is referential equality, and this is useMemo's job to take.

Another reason useCallback is if we actually need to create new function depending on the parameters. However this is probably not something we will encounter.
