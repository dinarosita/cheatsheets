React Hooks

Types of React Hooks

By React version 18, there are 15 hooks. With these 15 hooks we can achieve similar functionality to a class-based component.

useState: manage states

When there's certain state that applies within a rendered page, and the state can be changed around, hook useState is used to manage this. useState always return 2 values in an array format: the current state and a function to change the state. An initial state can be included as a parameter.

useEffect: perform side effects

When something is changed and that triggers something else to happen in respond of that, it's called side effect. Something that happen in respond of something. Hook useEffect function is used to handle this.

useMemo: memoize last values for next rendering

In a function component, everything gets rerendered including those that doesn't need to change. If we have a slow but rarely used function, it will get rerendered as well each time, causing every render becoming slow unnecessarily. In this situation, we use hook useMemo (memoization). It's caching the computed value. Hook useMemo will only recompute the memoized value when one of the deps (dependancies) has changed. Otherwise, it will use the memoized value.

useRef: Persist values between renders

useContext: Pass data and states through component tree

useReducer: Store and update states

useReducer is like useState, that it manages states and rerender component whenever the state changes. useReducer gives more concrete way to handle more complex function. useReducer has very similar pattern and use with redux, but it takes away a lot of boiler plate from redux.

useCallback: Cache function between rerendering. Like useMemo, but for functions.

Basics

Only call hooks from react function

React hooks can only be used inside a function component.

It can't be used in a class component. Class component already has their own thing to do what hooks do in function component.

Function vs class component:

  • Function component:
    function functionName () { ... }
  • Class component:
    class className extends React.component { ... }

Only call hooks at the top level

Read: Hooks rules

Everytime function runs, the hooks must be executed exactly the same order. Only call hooks at top level Meaning they can't be inside loops, conditions, or nested functions.

  • Correct:
    function functionName () { 
    useState()
    useState()
    useState()
}
  • Wrong:
    function functionName () { 
    if (this == something) {
        useState()
    }    
    useState()
    useState()
}

// React hook "useState" is called conditionally. React Hooks must be called in the exact same order in every component render.
