JavaScript: Promise

Promise

Producing code
Code that can take some time (imagine: the singer)
Consuming code
Code that must wait for the result (imagine: the fans)
Promise
Object that links producing code and consuming code
What is being promised? A result.
Syntax
new Promise(promiseFn)
Promise object constructor syntax
let myPromise = new Promise((myResolve, myReject) => {

    // "Producing code" (may take some time)
    myResolve();  // When succesful
    myReject();   // When error

});

// "Consuming code" (must wait for a fulfilled Promise)
myPromise.then(
    (value) => { /* code if succesful */  },
    (error) => { /* code if some error */ }
);
Producing code
When producing code obtains the result, it should call one of the two callbacks:
  • Result is success, call: myResolve(result value)
  • Result is error, call: myReject(error object)

Promise Object Properties

The Promise object supports 2 properties:
  • state
  • result
The 3 states and their results:
myPromise.state myPromise.result
"pending" undefined
"fulfilled" a result value
"rejected" an error object
Access
Promise properties state and result can't be accessed. Promise method must be used to handle this.
