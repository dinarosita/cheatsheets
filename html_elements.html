<h2 id="elements">HTML Elements</h2>

HTML elements are created using tags. Elements are specified further using attributes.

In example above:

  • Element: <h2 id="elements">HTML Elements</h2>
  • Tag: <h2>
  • Attribute: id
  • Attribute's value (of id): "elements"
  • (text content?): "HTML Elements"

Main root and its direct children

<main>
The root element. All other elements must be descendants of this element.
Directly under <main> element:
<head>
This is where metadata of the page is located. It's functional, not the displayed content.
<body>
Sectioning root.
This is where the actual displayed content is located, arranged in nesting elements each with role in the page composed by styling and organization.

Document metadata

<head>
Document's metadata elements are contained within (children of) <head> element.
<title>
Browser's title bar or page's tab. Text only. Tag within this element will be ignored.
Example:
<title>HTML Intro</title>
<meta>
For metadata that can't be represented by the rest of metadata elements.
Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<link>
To link the html with external resources and spevifies the relationship
Common attributes:
  • rel: relationship. Values example: stylesheet, preconnect.
  • ref: URL
Examples:
<link rel="stylesheet" href="./style.css">
<link rel="preconnect" href="https://fonts.googleapis.com">
<style>
Contain internal styling for the document. Better idea is using external styling in a separate css file instead and link it using <link> tag. The syntax is similar to the external css file.
Examples:
<style>
p {color: blue}
</style>
<base>
Specifies the base URL to use for all relative URLs in the document.
Only 2 attributes allowed:
  • href: URL of all the relative links in the page.
  • target: Target page of opening any page link if not specified further.
There should be just 1 declaration of each attributes. If there are more, only the first one is acknowledged.
Example:
<base href="https://duaransel.com/" target="_blank">
