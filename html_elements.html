<h2 id="elements">HTML Elements</h2>

HTML elements are created using tags. Elements are specified further using attributes.

In example above:

  • Element: <h2 id="elements">HTML Elements</h2>
  • Tag: <h2>
  • Attribute: id
  • Attribute's value (of id): "elements"
  • (text content?): "HTML Elements"

Semantic elements

Semantic elements
Semantic elements are elements with meaning
Examples: <header>, <article>, <form>, <nav>, and <table>
Non-semantic elements
Non-semantic elements tells nothing about its content.
Examples: <div> and <span>

Structure

<html> is the main root and its direct children are <head> where we put metadata and <body> where we put the content. Metadata elements inside <head> are written as direct children. The content elements though, are very flexible and can be super nesting. Below is just an example to give a rough idea.

Keep in mind that many semantic tags are actually from HTML5 in 2008. Before then, there were no <header>, <nav>, <section> and <footer> for example. They are all sectioned inside nesting <div> when needed.

<html>
    <head>
        <title>...</title>
        <meta>
        <link>
        <base>
        <style>...</style>
    </head>
    <body>
        <header>...</header>
        <nav>...</nav>
        <main>
            <section>...</section>
            <section>...</section>
            <section>...</section>
        </main>
        <aside>...<aside>
        <footer>...</footer>
        <script>...</script>
    </body>
</html>

In the following sections, elements will be listed by their tags and sectioned based on their functions/roles. Listing will be given in nesting style. Some parent-child relations are rather exact, some are more of an illustration or common example to give a better idea how these element tags are used.

Main root

<main>
The root element. All other elements must be descendants of this element.
Directly under <main> element:
<head>
This is where metadata of the page is located. It's functional, not the displayed content.
<body>
Sectioning root.
This is where the actual displayed content is located, arranged in nesting elements each with role in the page composed by styling and organization.

Document metadata

<head>
Document's metadata elements are contained within (children of) <head> element.
<title>
Browser's title bar or page's tab. Text only. Tag within this element will be ignored.
Example:
<title>HTML Intro</title>
<meta>
For metadata that can't be represented by the rest of metadata elements.
Examples:
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<link>
To link the html with external resources and spevifies the relationship
Common attributes:
  • rel: relationship. Values example: stylesheet, preconnect.
  • ref: URL
Examples:
<link rel="stylesheet" href="./style.css">
<link rel="preconnect" href="https://fonts.googleapis.com">
<style>
Contain internal styling for the document. Better idea is using external styling in a separate css file instead and link it using <link> tag. The syntax is similar to the external css file.
Examples:
<style>
p {color: blue}
</style>
<base>
Specifies the base URL to use for all relative URLs in the document.
Only 2 attributes allowed:
  • href: URL of all the relative links in the page.
  • target: Target page of opening any page link if not specified further.
There should be just 1 declaration of each attributes. If there are more, only the first one is acknowledged.
Example:
<base href="https://duaransel.com/" target="_blank">
<body>
<img src="img/placeholder.jpg">
<a href="tags/tag_base.asp">HTML base Tag</a>
</body>

Content sectioning

<body>
<header>
Introductory content, navigational aids, heading elements, logo, search form, author name, and other elements.
It can be function as the banner landmark, the header of the page, when the element is a direct child of <body> element.
<header>
<h1>Codey Luwak</h1>
<img src="..." alt="Luwak logo">
</header>
It can be the header of other sectioning elements like <article>, <aside>, <main>, <nav>, or <section> element.
<article>
<header>...</header>
<main>...</main>
</article>
<nav>
Navigation block.
Common examples are unordered list <ul> presented as a sidebar, navigation bar, or dropdown menu. But it actually doesn't have to be structured like that. For example, the link choices can be given within a paragraph <:p>
A document might have more than 1 <nav> element.
Usually link menu in the bottom of the page are given within <footer> instead.

Obsolete and depracated elements

Do not use. Replace if was used.

Sectioning
<dir>
Use instead: <ul>
<hgroup>
Use <h1>-<h6> directly
Section layout
<center>
Use CSS property instead.
For centering text: text-align: center
For centering block: margin-right: auto and margin-left: auto
<nobr>
Prevent text from automatically wrapping across multiple lines.
Use CSS property instead: white-space: nowrap
<plaintext>
Use instead: <pre>, <code>, or CSS font-family with monospace value.
<xmp>
Use instead: <pre>, <code>, or CSS font-family with monospace value.
<tt>
Teletype Text element
Use instead: <code>, <kbd>, <samp>, <var>, <pre>, or <span> styled with CSS font-family
Text role
<acronym>
Use instead: <abbr>
Text annotation
<rb> <rtc>
Check out more current <ruby> elements
Font
<basefont>
Use various CSS font properties instead
<font>
