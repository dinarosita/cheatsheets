Flex Containers

On the container: display: flex or display: inline-flex

Default properties on the container:

  • flex-direction: row
  • The items start from the start edge of the main axis.
  • All direct children will be flex items

Default properties of the flex items:

  • flex-grow: 0
  • flex-shrink: 1
  • The items will stretch to fill the size of the cross axis.
  • flex-basis: auto
  • flex-wrap: nowrap
<!--index.html-->

<div class="container">
<div>Item 1</div>
<div>Item 2</div>
</div>
/* style.css */

Growing and shrinking

flex is actually a shorthand for flex-grow, flex-shrink, and flex-basis.
The standard value is flex: 0 1 0%.
When declared only with 1 value, it's referring to the flex-grow:
flex: 1
Meaning: grow each child box equally and it is actually flex: 1 1 0%.

flex-grow

  • The value is the growth factor. It is a single number.
  • Default value is 0. That's why without declaration, flex item border will wrap arpund the content.
  • flex: 1 to each flex item div: they will grow equally, to the exact same size
  • flex: 2 to 1 item div and flex: 1 to the rest: that box will grow twice wider than each of the rest.

flex-shrink

  • Similar to flex-grow, but for shrinking.
  • Only applied when all flex items size is larger than the container.
  • Default value is 1. Meaning every flex item will be shrinking evenly further declaration.
  • Specify flex-shrink: 0 when you don't want a flex item to shrink

flex-basis

  • Sets the initial size of a flex item. Growing and shrinking start from this baseline.
  • Default value is auto, which mean will acknowledge width declaration of the flex item.
  • Remember flex: 1 means flex: 1 1 0, meaning we set it to 0, meaning we ignore the declared width. Meaning flex items will be growing or shrinking evenly according to the growth/shrink factor.
  • flex: 1 1 auto will take declared width into consideration.

Axes

In our regular English language, website orientation will be left to right just like our writing. This might not be the case for some other languages. Which one is the main axis will depend and the direction will depend on this. We limit this note to normal English use.

Axis:

  • Main axis: horizontal axis: row
  • Cross axis: vertical axis: column

flex-direction: column

  • Default direction for a flex container depends on the main axis. For regular English sites, it is horizontal (row).
    flex-direction: row
    flex-direction: row-reverse
  • To change it into vertical (column):
    flex-direction: column
    flex-direction: column-reverse
  • When the flex-direction is vertical, the flex-basis referred to height instead of width.

The consequence:

In vertical flexbox, it can be more complicated to define the flex values. An empty box (either flex-container or flex-item) doesn't have height by itself. It collapses. While the standard for the width, it fills the whole horizontal space. So in the vertical box, to avoid collapsing boxes, height need to be defined, and pay attention when using the flex: 1 shorthand because it is changing the flex-basis value from auto (acknowledging the height/width) into 0 (ignoring, resulting in the collapse of the empty boxes.)

Some good options:
  • Define flex item's height and:
    • use flex: 1 1 auto.
    • use flex-grow: 1 because it left standard flex-basis at auto
  • Define flex container's height and the usual shorthand flex: 1 can be used because now the container doesn't collapse vertically.

Alignment

Shorthand flex:1 is indeed handy.

width: 130px;
height: 80px;
flex: 1;
width: 130px;
height: 80px;
flex: 1;
width: 130px;
height: 80px;
flex: 1;

Note: in example above, cointainer's height is set to 100px and items height are not set. It is automatically fill in the vertical space. It's a standard behaviour. Even without flex:1, the items will fill in the vertical space too.

But how if we want to retain the declared size (width/height) of the flex-items? Here are some flex-container properties we can use:

  • justify-content for aligning items across the main axis (horizontal)
    • justify-content: space-between
      The flex-items will be distributed evently horixontally by stretching equal size spaces in between.
      In the next 3 examples, the container's height is declared to be 140px to illustrate the vertical allignment.
      height: 140px.
      width: 130px;
      height: 80px;
      flex: 1;
      width: 130px;
      height: 80px;
      flex: 1;
      width: 130px;
      height: 80px;
      flex: 1;
    • justify-content: center
      gap: 8px
      The flex-items will be centered horizontally with defined gap. If gap is not defined, the will stick to each other (if the items don't have declared margin either).
      width: 130px;
      height: 80px;
      flex: 1;
      width: 130px;
      height: 80px;
      flex: 1;
      width: 130px;
      height: 80px;
      flex: 1;
  • align-items for aligning items across the cross axis (vertical)
    • align-items: center
      The flex-items will be centered vertically.

      height: 80px;
      flex: 1;
      width: 130px;
      height: 80px;
