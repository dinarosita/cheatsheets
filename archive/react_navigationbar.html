Link component

Provided by react dom package

  • import { Link } from "react-router-dom";
  • The regular anchor tag "a href" link html element actually work, but has one big disadvantage. Upon clicking, new request will be sent to server. The server is the one hosting react application and the server will reply with our application and the router will figure out which page to render. So it will work too. However it will be redundant. We already are in our running react application. It will be great if we don't leave it just because we want to navigate somewhere. Also sometimes sending extra request is not something we neccesarily want to do.
  • Link component can be wrapped around our link text just like anchor tag and render our anchor tag in the end but internally react router dom attaches a click listener to the anchor tag and when we click on it, it will prevent browser default of sending request and instead parse the URL, change the browser url bar, but not sendign request, instead load the appropriate component onto the screen just with react and js. We stayed on the already loaded page. No extra request sent.
  • "to" prop for the path

App.js

  • Import MainNavigation component from components/layout
  • Add component above the switch statement (as self closing tag)
import { Route, Switch } from 'react-router-dom';

import AllMeetupsPage from './pages/AllMeetups';
import NewMeetupPage from './pages/NewMeetup';
import FavouritesPage from './pages/Favourites'; 

function App() {
    return (
        <div>
            <Switch>
                <Route path="/">
                    <AllMeetupsPage / exact={true}>
                </Route>
                <Route path="/new-meetup">
                    <NewMeetupPage />
                </Route>
                <Route path="/favourites">
                    <FavouritesPage. />
                </Route>
            </Switch>
        </div>
    );
}

export default App;

Rendering

  • import { Route } from 'react-router-dom';
    • The standard rendering of react: All correct parts of path will be rendered. Which mean child path will include the parent path:
    • localhost:3000
      • All Meetups Page
    • localhost:3000/favourites
      • All Meetups Page
      • Favourites Page
  • import { Route, Switch } from 'react-router-dom';
    • Often this is not desired. Use component Switch.Now only one will be rendered.
    • In the function, use component Switch to wrapp all the component Route
    • This will read from the top and render the first match. If it's the parent route at the top, it will be used.
    • localhost:3000/favourites
      • All Meetups Page
  • Props "exact" is added to prevent above
    • exact={true} or just exact
    • localhost:3000/favourites
      • Favourites Page
