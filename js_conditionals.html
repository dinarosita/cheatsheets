Comparisons

Comparisons using <, >, ==, ===, !=, !==, <=, and >= have been discussed a lot in the previous notes. Here's the summary.

  • Return boolean values
  • Strings are compared letter by letter just like in dictionary. Order is Unicode order. Case sensitive, with the uppercases comes before (smaller than) the lowercases as listed in the Unicode.
  • When values of different types are compared using non-strict equality, they are converted to numbers.
  • In strict equality, values of different types are never the same.
  • null == undefined and they don't equal to anything else
  • Be extra careful when comparing variables that the value might be null or undefined as they have specific behaviours.

Check out the complete reference here: JavaScript: Comparison

else

When it's more like conditional inclusions and exclusions, use if system.

switch

When it's more like options, especially when there are many, use switch.

The if Statement

if (condition 1) {
block of code 1;
} else if (condition 2) {
block of code 2;
} else {
block of code x;
}

It reads:

  • If condition 1 is true: Execute block of code 1. Exit.
  • Otherwise, if condition 2 is true: Execute block of code 2. Exit.
  • Anything else: Execute block of code x. Exit.

We can have more than have one else if.

Note that only exclusion from previous condition goes to the next. When a condition is met, the block of code for it is run, then it exits.

Conditions can be as many as needed. But in the case there are a lot of conditions (especially options type), consider using switch instead of if that's more like a list than inclusion-exclusion conditions.

if (time < 10) {
    greeting = "Good morning";
} else if (time < 20) {
    greeting = "Good day";
} else {
    greeting = "Good evening";
}
