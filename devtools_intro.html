DevTools

Introduction

DevTools: developer tools. Used to inspect elements.

DevTools tabs (panels):

  • Elements: View and change the DOM and CSS.
  • Console: View messages and run JavaScript from the console.
  • Sources: Debug JavaScript
  • Network: View and debug network activity.
  • Performance (Timeline): Find ways to improve load and runtime performance.
  • Memory (Profiles): Fix memory problems, JavaScript CPU profiler.
  • Application: Inspect all resources that are loaded.
  • Security: Debug mixed content issues, ceritificate problems, and more.
  • Lighthouse (Audit):
  • Recorder:

Open Chrome DevTools:

  • Ctrl+Shift+C for elements tab
  • Ctrl+Shift+J for console tab
  • Ctrl+Shift+I to open the last panel opened
  • From Chrome's main menu: Click Customize and control Google Chrome, select More Tools > Developer Tools.
  • Auto-open DevTools on every new tab:
    • On the command line: $ google-chrome --auto-open-devtools-for-tabs
    • Only works if there's no chrome already opened
    • Future opening html code with xdg-open file"name".html will open in new chrome window without DevTools pre-opened
    • Note, chrome opened from terminal keeps erroering. with or without the tags

Elements and Style

Element tab is for inspecting html elements and css style applied to each of the elements. It can make a quick change on the style and observe he effect. It can even copy the whole style of an element from different site and pastes in our chose element to see how it looks.

2 panes:

  • Elements pane
  • Styles pane

Access:

  • Press F12
  • Ctrl+Shift+c
  • In viewport, right click on element, choose inspect element.

Choosing an element:

  • In viewport, right click on element, choose Inspect element.
  • In elements pane, click on the element.
  • In elements pane, click Select an element, and click the element in the viewport.
  • In console pane, run a query like document.querySelector("p"). Right click the result and select Reveal in Elements panel

Styles tab in styles pane

  • View and change CSS
    • Changing just one particular element's style: Add property and value between element {...}.
    • Changing the group/class rule: Find the appropriate class/tag name to insert the property and value. FOr example: H2 {...}, .formula {...}, etc.
  • Add CSS declaration to an element: right click on element, choose inspect, modify styles.
  • .cls: To add a new class to an element
  • .hov: toggle to a CSS pseudo-class of an element such as :active, :focus, :hover, :visited, :focus-within, or :focus-visible.
  • Box model (at the bottom): Changing dimension of an element directly in box model is possible. It assumes px if written without unit.
  • Filter text box: Searching specific CSS properties or values.

Computed tab in styles pane

  • View only CSS that's actually applied to an element. They are listed in alphabetical order
  • Inherited propeties are shown in opaque mode. Check Show all box to see them all.
  • Filter text box: Searching specific CSS properties or values.

Command menu

  • Run>command panel
  • How: Ctrl+Shift+P
  • Start typing the keyword and choose from the selections displayed. Examples:
    • dock
    • screeshot

Some useful views:

  • View a page in print mode
    • Open command menu, start typing Rendering, select show rendering, go to Emulate CSS Media
  • View used and unused CSS with coverage Tab
    • Open command menu, start typing Coverage, select show coverage. In the coverage tab just opened, click the round arrow button Start Instrumenting Coverage and Reload Page.
    • The result shows: How much CSS and JS used from each file that the browser load. Green represented used CSS, red unused CSS.

Up/down keys shortcuts

  • To increase/decrease numeric value
  • Alt+Up: 0.1 increment
  • Up: 1 increment or 0.1 if the current value is between -1 and 1.
  • Shift+Up: 10 increment
  • Shift+Page+Up: Increment 100
  • All above works with Down too to decrease

Console

Main function: Logging code executions.

How? Add console commands wherever wanted within the code, it will log out what's asked in console panel.

The console panel is also a handy place to try out codes snippets.

Access:

  • F12 and choose console tab.
  • ctrl+Shift+j
  • Console in the drawer: esc
    This allows a console panel to stay open regardless which tab is chosen. Will come handy for debugging, to open source tab and console panel at the same time.

Handy Loggings

console.log(pets)

The regular console.log(), can log "comments", variables, or combinations (comma separated).

console.log({pets})

Object shorthand syntax

With curly brackets {} wrapping the object, it will log the : {objName: objValue}.

Examples:

  • Array {pets} into {pets: Array(3)}
  • Primitive string {myCat} into {myCat: "Kitten"}
console.table(pets)

console.table() log a table. Sorting based on the values is possible by clicking the table headers. In the case the variabe is a primitive, it's just displayed plainly like using console.log.

console.trace("Comment")

console.trace() outputs a stack trace to the console. Not sure what it means, read further.

Source

Main function: Debugging.

How?

  • Put debugger; on your code. When devtools find it, it will stop there.
  • Additional breakpoint can be added simply by clicking on the line number on the source panel.
  • More breakpoint:
    • Dome mutation breakpoint
      • Go to Elements tab, right click on the element chosen
      • Choose Break on, choose Attribute modifications for example.
    • Event Listener breakpoint
      • Easily jumping to particular event without even knowing where is it located in the code.
      • In the second pane of source tab, in the bottom, find Event listener breakpoint.
      • Then, for example, expand mouse and enable Click
  • Watch the journey of variable values along operation using life expression:
    • Open console in drawer(esc)
    • Click expression option (eye button) and enter in the variables (can be multiple).
  • Add logpoint
    • Right click in the source code in the line number, and choose logpoint. Tips: consider log {varName}
    • It's like console.log but without the need of putting the command in the code.
  • We can do stuff with the values/variables that get logged in the console by right click and choose Store as global variable.
  • If we don't want to pause at the debuggers anymore, we can right click on it and choose never pause here
